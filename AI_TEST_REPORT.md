## 8. 发现的问题

1. `.gitignore` 原先缺少 `.env.*.local`，已做最小修复。
2. `pnpm lint` 会进入 Next.js ESLint 初始化交互，说明项目尚未配置 ESLint。
3. 访谈提纲 AI 输出偶尔会包含说明性前言，不是纯提纲内容。
4. 访谈提纲 API 在 AI 失败时返回 `fallback=true` 和 `markdown=null`，页面会 fallback 到 mock；直接调用 API 的消费方需要知道这个约定。
5. mock 访谈提纲仍使用旧的“开场白/基础信息/核心问题/深度追问”结构，和最新 Prompt 中的“暖场/事实/过程/案例/困难”结构不完全一致。

## 9. 建议优先修复的问题

1. 增加 ESLint 配置，让 `pnpm lint` 能非交互运行。
2. 将访谈提纲 mock 结构同步到最新 Prompt 结构，保证无 Key 或 API 失败时体验一致。
3. 进一步约束访谈提纲 Prompt：要求“直接从二级标题开始，不要写前言、说明或客套话”。

## 10. 下一步优化建议

1. 为核心 API route 增加最小自动化测试，覆盖创建项目、生成选题、选择主选题、生成方案。
2. 为本地 `localStorage` 模块补充浏览器端回归测试，重点覆盖刷新后数据保留。
3. 将 AI 输出解析做得更稳：访谈提纲可尝试要求固定 Markdown 标题，并在前端自动裁剪模型前言。
4. 为 AI 服务增加轻量调用状态提示，例如“AI 生成 / mock fallback”，便于调试和演示。
