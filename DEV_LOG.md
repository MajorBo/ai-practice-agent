### 当前状态

* 项目已经从纯 mock 原型进入真实 AI MVP 阶段。
* 当前已完成完整主链路：

  * 项目创建
  * 选题生成

  * 访谈提纲

* 当前尚未实现用户登录、多用户权限、文件上传和正式部署。

### 验收结果

* 未配置 `DEEPSEEK_API_KEY` 时，系统可以 fallback 到 mock 数据。
* 配置 `DEEPSEEK_API_KEY` 后，选题生成可以调用真实 AI。
* 配置 `DEEPSEEK_API_KEY` 后，访谈提纲生成可以调用真实 AI。
* API 调用失败时，页面不会崩溃。
* 生成结果能够结合项目主题、实践地点、团队人数、实践时间和预期成果。
* 本地运行正常。
* Git diff 中未发现真实 API Key。

### 发现的问题

* 当前只有部分模块接入真实 AI，报告大纲和报告初稿仍主要依赖 mock 逻辑。
* AI 输出质量高度依赖 Prompt，需要继续优化。
* 选题生成需要进一步控制“过大、过空、不可执行”的问题。
* 访谈提纲需要进一步增强田野感、追问能力和材料获取能力。
* 当前尚未接入云端数据库，换设备或多人协作能力不足。
* 当前尚未部署到公网，外部用户无法直接访问测试。

### 下一步计划

1. 优化选题生成 Prompt。
2. 优化访谈提纲生成 Prompt。
3. 用真实实践主题进行多轮 AI 输出质量测试。
4. 在选题和访谈提纲稳定后，再逐步接入：

   * 调研方案真实 AI 生成
   * 访谈纪要真实 AI 生成
   * 报告大纲真实 AI 生成
   * 报告初稿真实 AI 生成
5. 之后接入 Supabase，实现云端数据保存。
6. 再接入用户登录、权限控制和 Vercel 部署。

### 阶段总结

本阶段的关键意义在于：项目不再只是一个可点击的 mock 原型，而是开始具备真实 AI 生成能力。

当前产品已经具备“从实践主题到选题与访谈提纲”的真实 AI 辅助能力，初步验证了 AI实践调研Agent 作为高校社会实践调研工作流工具的可行性。
